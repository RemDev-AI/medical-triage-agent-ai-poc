# ==========================================================
#
# Version pinée et stable, AVEC son suffixe local "+cpu" — ce
# suffixe n'est PAS cosmétique : c'est lui que vLLM inspecte au
# runtime (vllm/platforms/__init__.py, cpu_platform_plugin()) pour
# résoudre current_platform en CpuPlatform. Ce suffixe n'existe
# que sur les wheels réellement compilés avec VLLM_TARGET_DEVICE=cpu
# (cf. vllm/setup.py : `if VLLM_TARGET_DEVICE == "cpu": version +=
# "+cpu"`). Sans lui — ex: si pip installe silencieusement le
# wheel standard "0.25.1" depuis PyPI au lieu du wheel CPU-natif de
# wheels.vllm.ai/0.25.1/cpu, ce qui peut arriver avec deux index en
# présence sans version explicitement suffixée — current_platform
# ne résout AUCUNE plateforme (ni CUDA ni CPU) et
# create_engine_config() lève "RuntimeError: Device string must
# not be empty" au démarrage du moteur (observé en prod).
# Ajuster ce suffixe au besoin selon la dernière version stable
# disponible au moment du déploiement — vérifier
# https://docs.vllm.ai/en/latest/getting_started/installation/cpu/
# pour la version recommandée courante et son suffixe exact.
